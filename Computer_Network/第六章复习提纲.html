〇、链路层概述（铺垫）

运输类比（课件原例）：从普林斯顿到洛桑——豪华轿车（到机场）→ 飞机（到日内瓦）→ 火车（到洛桑）。游客 = 数据报，运输段 = 通信链路，运输方式 = 链路层协议，旅行社 = 路由算法。

基本术语 ★必背

  • 节点（node）：主机和路由器；
  • 链路（link）：连接相邻节点的通信信道（有线/无线/局域网）；
  • 帧（frame）：第 2 层数据包，封装数据报；
  • 链路层职责：把数据报从一个节点经链路传到物理相邻的节点；
  • 不同链路可以用不同链路层协议（如第一跳 WiFi、下一跳以太网），各自提供的服务也可能不同。

链路层服务 ★

  1. 帧封装、链路接入：数据报封装成帧（加头部尾部）；共享介质时做信道接入；帧头用 MAC 地址标识源/目的（≠ IP 地址！）；
  2. 相邻节点间可靠传输：低误码率链路很少用；无线链路误码率高才用；
  3. 流量控制：相邻收发节点间的节奏控制；
  4. 差错检测：信号衰减、噪声造成错误；接收方检测到错误后要求重传或丢帧；
  5. 差错纠正：接收方不靠重传直接定位并纠正比特错误；
  6. 半双工/全双工：半双工——两端都能发，但不能同时。

链路层在哪实现？★

  • 在每台主机中，实现于网卡（NIC，网络接口卡）或芯片上（以太网/WiFi 卡），实现链路层+物理层；经主机总线（PCI）接入；是硬件、软件、固件的组合；
  • 发送端：封装成帧、加差错检测位等；接收端：检错、提取数据报上交；
  • 补充：IEEE 802.2 LLC（逻辑链路控制）位于 MAC 之上，构成数据链路层的两个子层。

一、差错检测与纠错

1.1 基本框架 ★

  • EDC：差错检测与纠正位（冗余位）；D：受保护的数据（可含头部字段）；
  • 接收方收到 D' 和 EDC'，判断是否出错；
  • 差错检测不是 100% 可靠的★：协议可能漏掉少数错误；EDC 字段越大，检测/纠正能力越强。

1.2 奇偶校验 ★必考

单比特奇偶校验：

  • 加 1 个校验位；偶校验：让 1 的总个数为偶数；
  • 只能检测单比特（奇数个）错误，不能纠正。

二维奇偶校验 ★：

  • 把数据排成 i 行 j 列，每行、每列各加一个校验位；
  • 能检测并纠正单比特错误：出错行的行校验和出错列的列校验都会异常，交叉定位 → 翻转该位即纠正；
  • 也能检测（但不能纠正）两比特错误。

1.3 互联网校验和（回顾第三章）

  • 把段内容当 16 位整数序列求反码和（最高位进位回绕），取反放入校验和字段；
  • 接收方重算比对：不等 → 有错；相等 → 未检出错误，但仍可能有错（弱保护）。

1.4 循环冗余校验（CRC）★必考会算

  • 更强大的差错检测编码，实际中广泛应用（以太网、802.11 WiFi）；
  • D：d 位数据（视为二进制数）；G：生成多项式（位模式），r+1 位；
  • 目标：选 r 个 CRC 位 R，使 <D,R> 能被 G 模 2 整除；
  • 发送 <D,R> = D·2r XOR R；接收方用 G 去除 <D,R>：余数非零 → 检测到错误；
  • 能检测所有小于 r+1 位的突发错误 ★。
R = remainder( D·2r ÷ G )　（模 2 除法 = 按位 XOR，不借位不进位）
  • 课件例：D=101110，G=1001（r=3）→ D·2³ = 101110000，模 2 除以 1001 得余数 011 → 发送 101110011；
  • 帧数据 10111 → 实际传输帧 = 帧数据+余数 = 10111011；
  • 标准生成多项式（了解）：CRC-12、CRC-16（x¹⁶+x¹⁵+x²+1）、CRC-CCITT（x¹⁶+x¹²+x⁵+1）、IEEE 802（32 位）。

二、多址接入协议（多路访问协议）

2.1 问题背景 ★

两种链路：

  • 点对点：以太网交换机↔主机、拨号 PPP；
  • 广播（共享介质）：传统总线以太网、HFC 上行、802.11 无线、4G/5G、卫星——像"鸡尾酒会"共享空气。

多址接入问题：单一共享广播信道，两个以上节点同时传输 → 干扰/冲突（collision）；需要分布式算法决定谁何时发送，而且协调本身也只能用这条信道（没有带外信道）。

理想的多址接入协议（速率 R 的信道）★必背四条：

  1. 只有一个节点要发时，它能以全速率 R 发送；
  2. M 个节点要发时，每个平均得 R/M；
  3. 完全去中心化：无需专门协调节点、无需时钟/时隙同步；
  4. 简单。

2.2 三大类 MAC 协议 ★必背

类别思想例子
信道划分把信道切成小份（时隙/频段/码片），独占分配TDMA、FDMA
随机接入不划分，允许冲突，冲突后"恢复"ALOHA、时隙 ALOHA、CSMA、CSMA/CD、CSMA/CA
轮流节点轮流用，发得多的可以用更久轮询、令牌传递

2.3 信道划分：TDMA 与 FDMA ★

  • TDMA（时分多址）：按"轮"访问，每站每轮一个固定长度时隙；没用的时隙空闲（例：6 站只有 1、3、4 要发，时隙 2、5、6 浪费）；
  • FDMA（频分多址）：频谱划分频带，每站固定一个频带；没用的频带空闲；
  • 共同缺点：低负载时浪费——单个节点也只能用 1/N 的资源。

2.4 随机接入：ALOHA 家族 ★含效率数字

纯 ALOHA：

  • 帧一到就立即发送，不同步；
  • 在 t0 发送的帧会与 [t0−1, t0+1] 内发送的帧冲突 → 冲突危险区 = 2 个帧时；
  • 效率 = 1/(2e) ≈ 18% ★。

时隙 ALOHA（slotted ALOHA）：

  • 假设：等长帧、时间分等长时隙、只在时隙开始时发送、节点同步、冲突所有人可检测；
  • 操作：新帧在下一个时隙发送；冲突则之后每个时隙以概率 p 重传直到成功；
  • 冲突危险区缩小为 1 个帧时；
  • 优点：单活动节点可全速发、高度去中心化（只需时隙同步）、简单；缺点：冲突浪费、空闲时隙、需要时钟同步；
  • 效率：Np(1−p)N−1，N→∞ 时最大效率 = 1/e ≈ 37% ★。

2.5 随机接入：CSMA 与 CSMA/CD ★核心必考

CSMA（载波侦听多址）：发送前先听（"别打断别人"）——信道空闲才发，忙则推迟。

  • ★冲突仍会发生：传播时延使两个节点可能听不到对方刚开始的传输；冲突时整个分组传输时间被浪费；
  • 三种坚持策略（课件补充）：1-坚持（忙则持续监听，一空闲立即发）、非坚持（忙则随机延时后再听）、p-坚持（用于分时隙信道，空闲时以概率 p 发送）。

CSMA/CD（带冲突检测）：边发边听，检测到冲突立即中止，减少信道浪费（"礼貌的对话者"）；有线易实现、无线难。

以太网 CSMA/CD 算法 ★必背五步：

  1. NIC 收到数据报，创建帧；
  2. 侦听：空闲 → 发送；忙 → 等到空闲再发；
  3. 发送完整帧且无冲突 → 完成；
  4. 发送中检测到冲突 → 中止，发送阻塞（jam）信号；
  5. 中止后二进制指数退避：第 m 次冲突后从 {0,1,…,2m−1} 随机选 K，等待 K·512 比特时间回到第 2 步——冲突越多，退避越久。
  • 冲突检测方法（课件补充）：信号电平法、过零点检测法、自收自发比较法；
  • ★时隙长度：发送后经过 2τ（τ 为单程最大传播时延）没冲突即成功（1 km 同轴 τ≈5μs，2τ≈10μs）。
CSMA/CD 效率 = 1 ÷ (1 + 5·tprop/ttrans)

tprop→0 或 ttrans→∞ 时效率→1；比 ALOHA 好，且简单、廉价、去中心化。

2.6 无线场景与 CSMA/CA ★

为什么无线不用 CSMA/CD：① 边发边收成本太高；② 发送端检测不到接收端的冲突。

  • 隐藏站问题 ★：A、C 互相听不到（超出无线电范围），同时向 B 发送 → 在 B 处冲突；
  • 暴露站问题 ★：B 向 A 发送，C 听到 B 的信号就不敢向 D 发——其实并不会冲突。

CSMA/CA（冲突避免，802.11 用） ★：

  • 发送前监听：忙 → 二进制指数退避后继续监听，空闲再发；
  • 发送前要等帧间间隔（IFS）+ 退避时间；
  • 发送后不做冲突检测，靠接收方回 ACK 确认；没收到 ACK 判定丢失，再退避重传；
  • 与以太网区别：第 i 次退避从 22+i 个时隙中随机选。

2.7 轮流协议 ★

  • 动机：信道划分高负载好低负载差；随机接入低负载好高负载冲突多；轮流协议"两全其美"；
  • 轮询（polling）：主节点逐个"邀请"从节点发送；问题：轮询开销、延迟、主节点单点故障；
  • 令牌传递（token passing）：控制令牌依次传递，拿到令牌才能发；问题：令牌开销、延迟、令牌丢失单点故障（蓝牙、FDDI、令牌环使用）；
  • 实例：电缆接入网 DOCSIS——下行 FDM 广播；上行部分时隙分配（TDM）、部分随机接入竞争（二进制退避）——三类方法的混合体。

三、局域网（LAN）

3.1 MAC 地址 ★必背

  • 48 位（6 字节），固化在网卡 ROM（也可软件设置），十六进制表示，如 1A-2F-BB-76-09-AD；
  • 作用：在同一子网内把帧从一个接口送到另一个物理相邻接口（"本地"传输）；
  • 分配由 IEEE 管理，厂商购买地址块保证唯一；
  • 类比 ★：MAC 地址像身份证号（跟人走，扁平、可移植——换 LAN 不变）；IP 地址像邮政地址（跟位置走，层次化、不可移植——取决于所在子网）；
  • 广播地址：FF-FF-FF-FF-FF-FF。

3.2 ARP（地址解析协议）★必考流程

问题：知道目标的 IP 地址，如何查它的 MAC 地址？

  • 每个 IP 节点有 ARP 表：<IP 地址; MAC 地址; TTL>（TTL 通常 20 分钟）；
  • 流程（A 想发给同网段的 B）★三步：
    1. B 不在 A 的 ARP 表 → A 广播 ARP 查询（目的 MAC = FF-FF-FF-FF-FF-FF，含 B 的 IP），全网段都收到；
    2. B 单播 ARP 应答给 A（封装在普通以太网帧中，目的 MAC = A），告知自己的 MAC；
    3. A 把 B 的条目存入 ARP 表。

3.3 跨子网发送：IP 地址与 MAC 地址配合 ★经典必考

场景：A（111.111.111.111）经路由器 R 发给另一子网的 B（222.222.222.222）：

  1. A 创建 IP 数据报：源 IP=A，目的 IP=B（始终不变）；
  2. A 创建链路层帧：目的 MAC = R 的（左）接口 MAC（用 ARP 查），装入数据报；
  3. 帧到 R：R 解封装取出数据报上交 IP 层；
  4. R 查转发表确定出接口，重新封装：源 MAC = R 的（右）接口 MAC，目的 MAC = B 的 MAC；IP 源/目的不变；
  5. B 收到帧，解封装上交 IP 层。
IP 地址端到端不变，MAC 地址逐跳更换

3.4 以太网 ★必背

  • "主导"的有线局域网技术：最早广泛使用、简单便宜、速度从 10 Mbps 到 400 Gbps；
  • 物理拓扑：
    • 总线型（90 年代）：同轴电缆，所有节点同一冲突域；
    • 交换式（现在）：中心是交换机，每条链路独立跑以太网协议，节点间不冲突。

以太网帧结构 ★必背字段：

| 前同步码 | 目的 MAC | 源 MAC | 类型 | 数据(payload) | CRC |
  • 前同步码（preamble）：7 字节 10101010 + 1 字节 10101011，用于同步收发双方时钟；
  • 地址：6 字节源/目的 MAC；目的地址匹配或广播才上交，否则丢弃；
  • 类型（type）：指示上层协议（通常 IP），用于解复用；
  • CRC：接收方校验，错误则丢帧。

以太网特性 ★必考：

  • 无连接：收发网卡间无握手；
  • 不可靠：接收网卡不发 ACK/NAK，坏帧直接丢弃——靠上层（如 TCP）恢复，否则数据就丢了；
  • MAC 协议：非时隙 CSMA/CD + 二进制退避；
  • 802.3 标准家族：MAC 协议和帧格式相同，速率和物理介质不同（双绞线/光纤）。

3.5 以太网交换机 ★必考

  • 链路层设备：存储转发以太网帧，检查帧的 MAC 地址选择性转发；
  • 透明：主机感知不到交换机存在；即插即用、自学习：无需配置；
  • 每条链路专用、无冲突、全双工，A→A' 和 B→B' 可同时传输（但 A→A' 与 C→A' 不能同时——目的端口争用）。

交换表与自学习 ★必背：

  • 表项：(MAC 地址, 到达该主机的接口, 时间戳)；
  • 自学习：收到帧时记下（源 MAC → 进入的接口）。

过滤/转发算法 ★：

  1. 记录传入链路和源 MAC；
  2. 用目的 MAC 查表；
  3. 找到条目：若目的就在帧到达的网段 → 丢弃（过滤）；否则 → 从条目指示的接口转发；
  4. 没找到 → 泛洪（除来口外的所有接口）。

多台交换机互连：同样靠自学习，无需额外协议。

交换机 vs 路由器 ★必考对比：

交换机路由器
层次链路层设备（看帧头）网络层设备（看 IP 头）
转发表自学习/泛洪获得，按 MAC 转发路由算法计算，按 IP 转发
共同点都是存储转发，都有转发表

3.6 虚拟局域网（VLAN）★

动机：单一广播域的问题——① 扩展性：所有二层广播（ARP、DHCP、未知 MAC）要传遍整个 LAN；② 效率、安全、隐私；③ 管理：用户换楼但想留在原逻辑网络。

基于端口的 VLAN ★：

  • 支持 VLAN 的交换机可在一台物理交换机上定义多个虚拟局域网（如端口 1-8 = EE，9-15 = CS）；
  • 流量隔离：1-8 端口的帧只能到达 1-8——广播域被隔开；
  • 划分方式（课件补充）：按端口（静态）、按 MAC 地址（动态）、按 IP 地址（动态）；
  • VLAN 间转发要靠路由（就像两台独立交换机之间）；实际设备常是交换机+路由器一体；
  • VLAN 优点：便于管理虚拟工作组、安全（不同 VLAN 不能互相监听）、避免广播风暴、减少物理搬动。

跨交换机的 VLAN：802.1Q 与中继端口 ★：

  • 中继端口（trunk port）：在多台物理交换机之间传送各 VLAN 的帧；
  • 帧必须带 VLAN ID → 802.1Q 帧 = 普通以太帧插入 4 字节标签（2 字节标签协议标识符 0x8100 + 12 位 VLAN ID + 3 位优先级），并重算 CRC。

3.7 MPLS 与数据中心网络（了解）

  • MPLS（多协议标签交换）：用固定长度标签（而非最长前缀匹配）高速转发；标签交换路由器只看标签不看 IP；可做流量工程（同目的不同源走不同路）和快速重路由（预算备份路径）；MPLS 头插在以太帧头和 IP 头之间；
  • 数据中心网络：机架（20-40 刀片服务器）+ TOR（机架顶部）交换机 + 二级/一级交换机 + 边界路由器；多路径冗余互联提高吞吐和可靠性；负载均衡器做应用层路由；协议创新：RoCE、DCTCP（ECN）、SDN；
  • 一个网页请求的一天（综合复习线索）★：笔记本入网 → DHCP（拿 IP/网关/DNS）→ ARP（查网关 MAC）→ DNS（查 www.google.com 的 IP，UDP）→ TCP 三次握手 → HTTP 请求/响应 → 网页显示——把全书协议串成一条线。

4.3 易混概念辨析

概念对区别一句话
检错 vs 纠错检错：发现有错（重传或丢弃）；纠错：直接定位并改正（如二维奇偶校验纠单比特错）
单比特奇偶 vs 二维奇偶单比特：只能检奇数个错；二维：能检并纠单比特错
校验和 vs CRC校验和：16 位反码和，软件易实现但弱；CRC：模 2 除法，强，用于以太网/WiFi
纯 ALOHA vs 时隙 ALOHA纯：随到随发，危险区 2 帧时，18%；时隙：等时隙开头发，危险区 1 帧时，37%
CSMA vs CSMA/CD vs CSMA/CACSMA：先听后发；/CD：边发边听、冲突即停（有线以太网）；/CA：避免冲突+ACK 确认（无线 802.11）
为什么无线不用 CD边发边收代价大 + 发送端检测不到接收端的冲突（隐藏站）
隐藏站 vs 暴露站隐藏站：互相听不见 → 该让不让（冲突）；暴露站：听见了不相干的 → 该发不发（浪费）
MAC 地址 vs IP 地址MAC：48 位、扁平、可移植、链路层逐跳用；IP：32 位、层次化、随子网变、端到端不变
ARP vs DNSARP：IP → MAC（子网内广播查询）；DNS：域名 → IP（全球分布式数据库）
交换机 vs 路由器交换机：二层、看 MAC、自学习泛洪；路由器：三层、看 IP、路由算法
过滤 vs 转发 vs 泛洪过滤：目的就在来的网段→丢；转发：查表定向发；泛洪：查不到→除来口全发
总线以太网 vs 交换式以太网总线：一个冲突域、CSMA/CD；交换式：每链路独立冲突域、可同时多对传输
VLAN 内 vs VLAN 间VLAN 内：二层交换；VLAN 间：必须经路由器（三层）

4.4 一句话带过（非重点，了解即可）

  • MPLS：固定标签快速转发，支持流量工程与快速重路由。
  • 数据中心网络：TOR/多级交换/多路径/负载均衡器；RoCE、DCTCP、SDN。
  • DOCSIS：电缆网混合使用 FDM + TDM + 随机接入。
  • 蜂窝标准演进：1G 模拟语音 → 2G 数字（GSM）→ 3G → 4G → 5G（高速率低时延大连接）→ 6G（星地融合）。
  • "网页请求的一天"：DHCP → ARP → DNS → TCP 握手 → HTTP，是把全书串起来的复习主线。
