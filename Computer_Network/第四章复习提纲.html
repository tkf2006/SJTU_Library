一、控制平面与数据平面概念

1.1 网络层在做什么（铺垫）

  • 网络层把传输层的报文段从发送主机送到接收主机：发送方封装成数据报（datagram）交给链路层；接收方把段上交传输层；
  • 每个互联网设备（主机和路由器）都运行网络层协议；
  • 路由器：检查经过它的所有 IP 数据报的头部字段，把数据报从输入端口移到输出端口。

1.2 两大功能：转发 vs 路由 ★必背

转发（forwarding）路由（routing）
类比通过单个立交桥/路口规划从源到目的的整个行程
做什么把分组从路由器输入链路移到正确的输出链路确定分组从源到目的所走的路径
工具转发表路由算法

1.3 数据平面 vs 控制平面 ★本章核心必背

数据平面（data plane）控制平面（control plane）
范围本地的、每台路由器（per-router）的功能网络范围（network-wide）的逻辑
任务决定到达输入端口的数据报如何转发到输出端口决定数据报在源-目的路径上如何在路由器间路由
实现硬件，纳秒级软件，毫秒级
对应功能转发路由

控制平面的两种实现方法 ★：

  1. 传统路由算法：路由算法组件实现在每台路由器中，各路由器相互交互；
  2. 软件定义网络（SDN）：由（远程）控制器计算并把转发表安装到路由器中。

1.4 网络服务模型 ★

  • 互联网网络层提供的是"尽力而为（best effort）"服务：不保证 ① 数据报成功送达；② 送达的时间或顺序；③ 端到端最小带宽；
  • 对比：ATM 网络提供恒定比特率（CBR）等有 QoS 保证的服务；互联网也有 IntServ（RFC 1633）/DiffServ（RFC 2475）扩展；
  • 为什么"尽力而为"也成功了？机制简单易部署、带宽充足时"足够好"、CDN 等应用层服务贴近用户、弹性服务的拥塞控制有帮助。

二、路由器架构

2.1 总体结构 ★必背

路由器 = 输入端口 + 高速交换结构 + 输出端口（数据平面，硬件，纳秒级）+ 路由处理器（控制平面，软件，毫秒级）

2.2 输入端口 ★

处理流水线：物理层（比特级接收）→ 链路层协议（如以太网）→ 查找/转发/排队。

  • 去中心化交换：用头部字段值在输入端口内存中的转发表副本查找输出端口（"匹配 + 动作"），目标是以线速完成；
  • 两种转发方式 ★：
    • 基于目的地的转发：仅根据目的 IP 地址转发（传统）；
    • 通用（广义）转发：根据任意头部字段集合转发；
  • 输入端口排队：数据报到达速度超过进入交换结构的速度时排队。

2.3 最长前缀匹配 ★必考（会做题）

查转发表时，使用与目的地址匹配的最长地址前缀对应的表项。
目标地址范围（前缀）链路接口
11001000 00010111 00010*** ********0
11001000 00010111 00011000 ********1
11001000 00010111 00011*** ********2
否则3
  • 地址 11001000 00010111 00011000 10101010 同时匹配第 2、3 行 → 取最长的（第 2 行）→ 接口 1；
  • 地址 11001000 00010111 00010110 10100001 → 匹配第 1 行 → 接口 0；
  • 硬件实现：TCAM（三态内容寻址存储器）——不管表多大，一个时钟周期内完成检索（Cisco Catalyst 约 100 万条表项）。

2.4 交换结构（switching fabrics）★三种类型必背

任务：把分组从输入链路传到正确的输出链路；交换速率以线路速率倍数衡量，N 个输入理想需要 N×R。

类型原理瓶颈
经内存交换第一代路由器，CPU 直接控制，分组复制进系统内存内存带宽（每个分组过 2 次总线）
经总线交换输入端口内存 → 共享总线 → 输出端口内存总线带宽（总线争用），如 Cisco 5600 总线 32 Gbps，够接入路由器用
经互连网络交换纵横（crossbar）、Clos 网络、多级交换机；把数据报切成固定长度信元并行交换，出口重组扩展性最好；Cisco CRS：8 个交换平面，每平面 3 级互连，可达 100 Tbps

2.5 输入排队与 HOL 阻塞 ★

  • 交换结构比输入端口慢 → 输入队列排队 → 缓冲区溢出则丢包；
  • 队头阻塞（HOL blocking）：队首分组（在等争用的输出端口）挡住了队列中本可转发的后续分组。

2.6 输出端口排队与缓冲 ★

  • 数据报从交换结构到达的速率超过输出链路传输速率时需要缓冲；缓冲区满 → 按丢弃策略丢包（拥塞、缓冲不足是丢包原因）；
  • 调度规则决定下一个发送哪个排队分组（优先级调度涉及"谁获得最佳性能"与网络中立性）；
  • 缓冲多大？★
    • 经验法则（RFC 3439）：缓冲 = 典型 RTT × 链路容量 C（如 RTT 250ms、C=10Gbps → 2.5 Gbit）；
    • 较新建议：N 条流时，缓冲 = RTT·C / √N；
    • 缓冲过多会增加时延（实时应用差、TCP 响应迟钝）。

2.7 缓冲区管理与分组调度 ★

缓冲区管理：

  • 丢弃策略：尾部丢弃（丢新到的）或按优先级丢；
  • 标记：标记分组以指示拥塞（ECN、RED）。

分组调度四种 ★必背：

调度规则
FCFS（先来先服务/FIFO）按到达顺序发送
优先级调度按类分队（任何头部字段可作分类依据），先发最高优先级队列（队内 FCFS）
轮询（RR）服务器循环扫描各类队列，轮流给每类发一个分组
加权公平队列（WFQ）广义轮询，每类有权重 wi，每周期获得 wi/Σwj 比例的服务——每类有最低带宽保证
补充（了解）：网络中立——2015 美国 FCC 三条规则：不得阻止（no blocking）、不得限速（no throttling）、不得付费优先（no paid prioritization）。

三、网络协议（IP）

3.1 互联网网络层的组成

  • IP 协议：数据报格式、寻址、分组处理惯例；
  • ICMP 协议：错误报告、路由器"信令"；
  • 路由协议/SDN 控制器（控制平面，下一章）→ 生成转发表。

3.2 IPv4 数据报格式 ★必背重点字段

字段说明
版本（ver）IP 协议版本号
头部长度头部字节数
服务类型（ToS）diffserv（0:5 位）+ ECN（6:7 位）
总长度整个数据报字节数（最大 64KB，通常 ≤1500 字节）
16 位标识、标志、片偏移用于分片/重组
TTL（time to live）剩余最大跳数，每经过一个路由器减 1，为 0 丢弃
上层协议交付给 TCP 还是 UDP 等
头部校验和只校验头部
32 位源 / 目的 IP 地址寻址
选项如时间戳、记录路由（可选）
★常考开销：20 字节 TCP 头 + 20 字节 IP 头 = 40 字节（不含应用层开销）

3.3 IP 分片与重组 ★必考计算

  • 链路有 MTU（最大传输单元）——链路层帧能承载的最大数据量；不同链路 MTU 不同；
  • 大 IP 数据报在网络中分片（fragment），只在最终目的地重组，靠头部的标识（ID）、标志（fragflag）、偏移（offset）字段。

经典例题：4000 字节数据报（含 20 字节头，数据 3980 字节），MTU = 1500：

片lengthIDfragflagoffset
11500（数据 1480）x10
21500（数据 1480）x1185（=1480/8）
31040（数据 1020）x0（最后一片）370（=2960/8）
记忆要点：每片数据 = MTU − 20；offset 按 8 字节为单位；最后一片 fragflag = 0。

3.4 IP 地址与接口 ★

  • IP 地址：与每个主机或路由器接口关联的 32 位标识符；
  • 接口：主机/路由器与物理链路之间的连接——路由器通常多个接口，主机通常 1~2 个（有线以太网、无线 802.11）；
  • 点分十进制：223.1.1.1 = 11011111 00000001 00000001 00000001。

3.5 子网（subnet）★必背

  • 子网 = 共享相同 IP 地址前缀、不经过路由器就能物理互达的设备接口集合；
  • IP 地址结构 = 子网部分（高位）+ 主机部分（低位）；
  • 识别子网的诀窍：把每个接口从主机/路由器上剥离，形成的"孤岛"就是一个个子网；
  • 记法：223.1.1.0/24 —— "/24" 表示高 24 位是子网部分（子网掩码）。

3.6 分类 IP 地址（A/B/C/D/E）★必背

A00~12712816777216−2大型网络

私有地址（IANA 保留，仅限内网）★：A 类 10.0.0.0–10.255.255.255；B 类 172.16.0.0–172.31.255.255；C 类 192.168.0.0–192.168.255.255。

3.7 子网掩码 ★必考计算

  • 作用：指明地址中哪部分是网络+子网、哪部分是主机；掩码的网络+子网部分全 1，主机部分全 0；
  • 默认 C 类掩码：255.255.255.0；
  • 例：C 类地址 202.120.3.99、借 3 位作子网号（子网 011）→ 掩码 255.255.255.224（11100000）；
  • 简洁记法 202.120.3.99/27（27 = 掩码中 1 的个数）。
网络号（含子网）= IP 地址 ∧（按位与）掩码

3.8 CIDR（无类域间路由）★必背

  • 动机：B 类太大、C 类太小、地址紧缺；
  • 做法（RFC 1519）：任意长度的子网前缀，格式 a.b.c.d/x（x = 前缀位数）；把若干连续 C 类地址捆绑成一个分配单元；
  • 好处：提高地址利用率、缩短路由表（路由聚合）；
  • ★匹配规则：掩码中 1 位数最多的优先（即最长前缀匹配）。

经典例（剑桥/爱丁堡/牛津，从 194.24.0.0 分配）：

学校地址数范围掩码
剑桥2048194.24.0.0 ~ 194.24.7.255255.255.248.0 (/21)
爱丁堡1024194.24.8.0 ~ 194.24.11.255255.255.252.0 (/22)
牛津4096194.24.16.0 ~ 194.24.31.255255.255.240.0 (/20)

判断 194.24.17.4 属于谁：与各掩码按位与 → 与牛津掩码相与得 194.24.16.0 = 牛津的网络号 → 属牛津。

3.9 层次寻址与路由聚合 ★

  • ISP 拿到地址块 200.23.16.0/20，可以切成 8 块 /23 分给 8 个组织；
  • 路由聚合：ISP 对外只需通告一条 "200.23.16.0/20"，外界不必知道内部 8 个组织；
  • 更具体路由：组织 1 搬到另一 ISP 后，新 ISP 额外通告 "200.23.18.0/23"——因为最长前缀匹配，流量会正确流向新 ISP。

3.10 怎么获得 IP 地址？

主机获得地址：手工配置，或 DHCP（动态主机配置协议）★：

  • 目标：主机加入网络时动态从服务器获取 IP；可续租、地址可复用、支持移动用户；"即插即用"；
  • 四步（必背）：DHCP discover（广播：有 DHCP 服务器吗？可选）→ DHCP offer（服务器：给你这个 IP，可选）→ DHCP request（我要用这个 IP）→ DHCP ack（好，归你了）；
  • DHCP 不只给 IP，还能返回：第一跳路由器地址、DNS 服务器名称和 IP、网络掩码★；
  • 封装：DHCP 报文 → UDP → IP → 以太网（广播帧 FFFFFFFFFFFF）。

网络获得地址块：从其 ISP 的地址空间分配；ISP 从 ICANN（经 5 个地区注册机构）获得；IPv4 地址 2011 年已分配完 → NAT 和 IPv6 缓解。

3.11 NAT（网络地址转换）★必考

  • 思想：本地网络的所有设备对外共享一个 IPv4 地址；内部用私有地址（10/8、172.16/12、192.168/16）；
  • 优势：① ISP 只需给一个 IP；② 内部改地址不用通知外界；③ 换 ISP 不用改内部地址；④ 内部设备不可被外界直接寻址（安全）；
  • NAT 路由器三件事 ★必背：
    1. 传出数据报：把（源 IP，源端口）替换为（NAT IP，新端口）；
    2. 在 NAT 转换表中记住这对映射；
    3. 传入数据报：按表把（NAT IP，新端口）改回（原 IP，原端口）；
  • 例：10.0.0.1:3345 → 经 NAT 变成 138.76.29.7:5001 → 回复到 138.76.29.7:5001 → 查表改回 10.0.0.1:3345；
  • 争议：路由器"应该"只处理到第 3 层、违反端到端原则（操纵端口号）、地址短缺本应由 IPv6 解决、NAT 穿透问题；但 NAT 已无处不在（家庭、机构、4G/5G）。

3.12 IPv6（了解为主）

  • 动机：32 位 IPv4 地址耗尽；附加动机：40 字节定长头部加速处理、支持"流"的差异化处理；
  • 头部字段：版本、优先级、流标签、载荷长度、下一个头部、跳数限制、128 位源/目的地址；
  • ★与 IPv4 相比没有的：校验和（加快路由器处理）、分片/重组（不允许中途分片）、选项；
  • 隧道（tunneling）★：IPv6 数据报作为 IPv4 数据报的载荷穿过 IPv4 路由器区域（"包中包"封装）——因为无法让所有路由器同时升级（无"flag day"）；
  • 采用情况：Google 约 30% 客户经 IPv6 访问；25 年仍在部署中。

四、通用转发机制（广义转发与 SDN）

4.1 从"基于目的地"到"通用转发" ★必背

  • 回顾："匹配 + 动作（match plus action）"抽象：匹配到达分组的比特，执行相应动作；
  • 基于目的地的转发：只匹配目的 IP 地址，动作只有"转发到某端口"；
  • 通用（广义）转发：
    • 匹配：任意多个头部字段（链路层、网络层、传输层）的模式值；
    • 动作：丢弃（drop）、转发（forward）、修改（modify）分组，或发送给控制器；
    • 优先级：消除重叠模式的歧义；计数器：统计字节数和分组数；
  • 转发表此时称为流表（flow table）；流（flow）= 由头部字段值定义的一组分组。

4.2 SDN（软件定义网络）背景（了解）

  • 传统：单体路由器，专有硬件+专有系统（Cisco IOS），功能靠各种中间盒（防火墙、负载均衡、NAT 盒）；
  • SDN 类比"大型机 → PC 革命"：垂直整合封闭专有 → 水平开放接口、快速创新。

4.3 OpenFlow 流表 ★会读会写

可匹配的头部字段（跨三层）：入端口、源/目的 MAC、以太网类型、VLAN、IP ToS、IP 协议、源/目的 IP、TCP/UDP 源/目的端口。

四种动作：① 转发到端口；② 丢弃；③ 修改头部字段；④ 封装转发给控制器。

经典例子（必会判断）：

需求匹配动作
目的 IP 51.6.0.8 的数据报走 6 号端口IP Dst = 51.6.0.8，其余 *forward(port6)
阻止所有 TCP 22 端口（ssh）TCP d-port = 22，其余 *drop
阻止主机 128.119.1.1 发出的所有数据报IP Src = 128.119.1.1，其余 *drop
目的 MAC 22:A7:23:11:E1:02 的帧走 3 号口MAC dst = 该地址forward(port3)

4.4 匹配+动作如何统一各种设备 ★常考对应关系

设备匹配动作
路由器最长目的 IP 前缀转发到一条链路
交换机目的 MAC 地址转发或泛洪
防火墙IP 地址 + TCP/UDP 端口号允许或拒绝
NATIP 地址 + 端口重写地址和端口
  • SDN 控制器编排（orchestrate）多台交换机的流表，可实现全网级行为（如"h5、h6 的流量必须经 s1 再到 s2"）；
  • 网络可编程性：今日更通用的编程语言 P4。

4.5 中间盒与互联网架构原则（了解）

  • 中间盒（middlebox，RFC 3234）：源和目的之间路径上执行非标准 IP 路由器功能的设备——NAT、防火墙/IDS、负载均衡器、缓存等，无处不在；趋势：白盒硬件 + SDN 集中控制 + NFV（网络功能虚拟化）；
  • IP 沙漏（细腰）：物理/链路层协议很多，传输/应用层协议很多，但网络层只有一个 IP——每台互联网设备都必须实现的"细腰"；
  • 互联网三大架构信念（RFC 1958）：简单连通性、IP 是细腰、智能在端系统（边缘）而非网络内部；
  • 端到端原则：某些功能（如可靠传输）只有在通信两端的应用知晓与帮助下才能完全正确实现 → 不应放在低层/网络内部。

5.3 易混概念辨析

概念对区别一句话
数据平面 vs 控制平面数据平面：本地、每路由器、转发、硬件纳秒级；控制平面：全网、路由、软件毫秒级
转发 vs 路由转发 = 过一个路口（查表）；路由 = 规划全程（算法）
传统控制平面 vs SDN传统：路由算法分布在每台路由器里；SDN：远程控制器统一计算并下发转发表
基于目的地转发 vs 通用转发前者只看目的 IP、只会"发往某口"；后者匹配任意字段、可丢弃/修改/送控制器
输入排队 vs 输出排队输入排队：交换结构太慢（伴随 HOL 阻塞）；输出排队：输出链路太慢
直接广播 vs 有限广播直接：网络 ID + 全 1，向指定网络广播；有限：32 位全 1，只在本地网广播
分类地址 vs CIDR分类：前缀长度固定（A/B/C）；CIDR：前缀任意长（a.b.c.d/x），提高利用率、缩短路由表
IPv4 vs IPv6v6：128 位地址、40 字节定长头、无校验和、无分片、无选项，有流标签
NAT 表方向出站：换源（IP+端口）并记表；入站：查表换回目的（IP+端口）
路由器/交换机/防火墙/NAT（匹配+动作视角）分别匹配：IP 前缀 / 目的 MAC / IP+端口 / IP+端口；动作：转发 / 转发或泛洪 / 允许或拒绝 / 重写

5.4 一句话带过（非重点，了解即可）

  • 网络中立：no blocking、no throttling、no paid prioritization（2015 FCC）；ISP 按"电信服务"还是"信息服务"监管影响重大。
  • IPv6 过渡：隧道封装"IPv6 in IPv4"；部署 25 年仍在进行。
  • 中间盒/NFV/P4：网络功能从专有硬件走向白盒硬件上的可编程软件。
  • 端到端原则与 IP 沙漏：智能在边缘，网络层保持简单统一（只有 IP 一个协议）。
